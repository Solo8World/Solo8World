  • 提交类型：

    • feat：新功能开发
    • fix：bug修复
    • docs：项目文档改动
    • style： 代码格式（不影响代码运行的变动,空格,格式化,等等）
    • refactor：重构（理论上不影响功能的代码重构）
    • perf: 性能优化
    • test：增加或者修改测试
    • build: 影响构建系统或外部依赖项的更改(maven)
    • ci: 对CI配置文件和脚本的更改
    • chore：对非 src 和 test 目录的修改
    • revert: 还原上一次提交
